# Contributing to CmdBook

Thank you for your interest in contributing to CmdBook. Contributions are welcome and appreciated. This document provides guidelines to help you get started and ensure a smooth collaboration process.

---

## Getting Started

1. Fork the repository.
2. Clone your fork to your local machine.
3. Create a new branch for your changes.

```bash
git clone https://github.com/arifinsiddiqzisan/cmdbook.git
cd cmdbook
git checkout -b feature/your-feature-name
```

---

## How to Contribute

You can contribute in several ways:

* Reporting bugs
* Suggesting new features
* Improving documentation
* Submitting code improvements or fixes

---

## Reporting Issues

If you find a bug or have a feature request:

1. Check if the issue already exists.
2. If not, open a new issue.
3. Provide a clear and descriptive title.
4. Include steps to reproduce (for bugs).
5. Attach screenshots or logs if applicable.

---

## Code Contributions

Before submitting code:

* Ensure your code is clean and readable.
* Follow the existing project structure and naming conventions.
* Write meaningful commit messages.

Example:

```bash
git commit -m "Add command search functionality"
```

---

## Pull Request Process

1. Push your changes to your fork.
2. Open a Pull Request (PR) to the main repository.
3. Clearly describe what your PR does.
4. Link related issues if applicable.
5. Be open to feedback and make necessary changes.

---

## Code Style Guidelines

* Keep code simple and maintainable.
* Use consistent naming conventions.
* Avoid unnecessary complexity.
* Add comments where needed.

---

## Documentation

* Update documentation if your changes affect usage.
* Keep explanations clear and concise.

---

## Respect and Conduct

Please follow the project's Code of Conduct. Be respectful, constructive, and collaborative in all interactions.

---

## Need Help?

If you have any questions, feel free to open an issue or contact the maintainers.

---

Thank you for contributing to CmdBook.
